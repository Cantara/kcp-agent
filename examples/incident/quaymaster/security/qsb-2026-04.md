# QSB-2026-04 — Remote code execution in Broker management listener

*(Fictional example bulletin.)*

**Affected:** Quaymaster Broker 4.4.0 through 4.8.1 with the management
listener enabled (the default).

**Fixed in:** 4.8.2.

**Detail:** a crafted, unauthenticated request to the management listener
can cause the broker to execute attacker-supplied configuration directives,
leading to remote code execution as the broker service account.

**Workaround until you can patch:** disable the management listener or
restrict its port to trusted management hosts.

**Detection:** unauthenticated POSTs to `/mgmt/apply` in listener logs;
unexpected broker child processes.
